# ライブ通訳サービス（事業版）企画書

作成日: 2026-07-11  
工程: 企画  
状態: Asaph方針確認済み・商品名確定

## 1. 企画概要

ShalomWorksが運営し、Asaph個人が販売者となる、一般公開の会員制リアルタイム音声通訳サービスを開発する。利用者はブラウザから入力言語と出力言語を選び、マイクへ話すことで、翻訳された音声を低遅延で聞くことができる。

サービスは前払いの通訳時間制とし、メール認証を完了した新規会員には初回15分を無料付与する。LP、会員機能、決済、通訳、利用時間管理、管理画面を一つの事業サービスとして提供する。

商品名は「Live Interpreter」とする。運営ブランドは「ShalomWorks」とする。

## 2. 運営と販売主体

- 運営ブランド: ShalomWorks
- 販売者: Asaph個人（法定氏名は本番の非公開設定・法定表示で入力）
- 売上受取: Asaph個人名義のStripeアカウントおよび登録銀行口座
- 献金: 本サービスの機能・表示・会計管理の対象外
- LP、利用規約、決済画面では献金について記載しない

## 3. 解決する課題

- 通訳者を用意できない場面でも、話し続けながら別言語の音声を届けたい。
- 専用アプリをインストールせず、スマートフォンやPCですぐ利用したい。
- APIキーを持たない一般利用者でも、料金を事前に把握して安全に使いたい。
- 教会、イベント、教育、旅行、接客など多様な場面で言語の壁を下げたい。

## 4. 想定利用者

国籍・人種を問わず、OpenAI Realtime Translation APIの対応言語を使う世界中の利用者を対象とする。

初期の主要利用場面:

- 教会の礼拝、集会、案内
- 小規模イベントやセミナー
- 教育、国際交流
- 旅行、生活案内
- 店舗、施設、オンライン接客

## 5. 提供価値

- ブラウザだけで利用開始できる
- 話し続けながら音声から音声へ通訳する
- 入力音量を常時確認できる
- 通訳音声と同程度の音量を事前テストできる
- 残り通訳時間を分かりやすく表示する
- 会話内容を原則保存しない
- 日本語、英語、中国語のUIを初期提供する

## 6. 事業モデル

### 無料体験

- メール認証完了後に15分付与
- 1人1回を原則とする
- 登録後30日で失効
- 譲渡、換金、払い戻し不可

### 有料商品案

| 商品 | 通訳時間 | 価格案（税込表示は税務確認後確定） |
|---|---:|---:|
| Starter | 60分 | 1,500円 |
| Standard | 300分 | 5,500円 |
| Event | 1,000分 | 15,000円 |

内部では秒単位で残高を保持し、利用者には分・秒で表示する。初期は日本円の固定商品とし、Stripe Checkoutによる一回払いを採用する。

## 7. 初期リリース範囲

- 多言語LP
- 会員登録、メール認証、ログイン、パスワード再設定
- 初回15分無料付与と不正取得対策
- 通訳時間の購入、残高、購入履歴
- 入力言語、出力言語、音声の選択
- 常時マイクレベル表示
- 通訳後音声の音量テスト
- リアルタイム音声通訳
- 秒単位の利用時間差し引きと残高0での停止
- 利用履歴
- 管理画面
- Stripe決済とWebhook
- 特定商取引法、利用規約、プライバシー等の法定ページ
- LP用実演動画の掲載枠

## 8. 初期対象外

- 字幕、文字起こし表示
- 音声・翻訳本文の保存
- 会員間の時間譲渡
- 時間の換金
- サブスクリプション
- 法人向け請求書払い
- Zoom、Teams等への直接組み込み
- 専門用語辞書、聖書翻訳引用支援
- ネイティブアプリ

## 9. システム方針

- LP、会員、決済、管理画面: ConoHa WING
- DB: MySQL
- 決済: Stripe Checkout + 署名検証Webhook
- 通訳: OpenAI Realtime Translation API
- リアルタイム制御: 残高を強制できる中継サーバー
- ソース: GitHub非公開リポジトリ
- デプロイ: GitHub Actions
- 秘密情報: GitHub Secretsとサーバー非公開設定のみ。Gitへ保存しない

## 10. 成功条件

- 新規会員がメール認証後、15分無料で通訳できる
- 有料時間を購入すると残高へ一度だけ正しく反映される
- 残高以上に利用できず、0秒で通訳が停止する
- 会話の音声・本文を保存しない
- PCとスマートフォンの主要ブラウザで利用できる
- LPから無料登録、体験、購入まで迷わず進める
- 管理者が売上、API原価、残高、利用量、不正兆候を確認できる
- 決済、API、DBの秘密情報がクライアントへ露出しない

## 11. 主要リスク

| リスク | 対応 |
|---|---|
| 無料枠の大量取得 | メール認証、CAPTCHA、IP・端末・頻度による制限 |
| クレジット超過利用 | ブラウザ任せにせず中継サーバーで残高を強制 |
| 二重付与 | StripeイベントIDの一意制約とDBトランザクション |
| API原価の急増 | 原価監視、利用上限、異常セッション停止 |
| 誤訳 | AI通訳であることと重要判断への利用注意を明示 |
| 法令・税務 | 販売開始前に特商法、前払式支払手段、消費税を専門家確認 |
| 個人情報流出 | 最小収集、暗号化通信、秘密分離、ログ制限 |
