# ライブ通訳サービス（事業版）要件定義書

作成日: 2026-07-11  
工程: 要件定義  
状態: 初版確定・法定情報と商品名は入力待ち

## 1. サービス要件

- 一般公開された会員制Webサービスとする。
- 運営表示はShalomWorks、販売者はAsaph個人とする。
- 利用者は会員登録、無料体験、時間購入、通訳をブラウザ内で完結できる。
- 通訳本文、音声、文字起こしは原則保存しない。
- 対応言語はAPIが公式に対応する範囲だけを表示する。

## 2. 利用者と権限

| ロール | 権限 |
|---|---|
| 未登録者 | LP、料金、対応言語、FAQ、法定ページの閲覧 |
| 会員 | 通訳、購入、残高・履歴・設定・退会 |
| サポート | 会員・決済・セッションの閲覧、停止。残高変更不可 |
| 管理者 | 全管理機能、手動付与、返金処理、設定変更、監査閲覧 |

管理者には二要素認証を必須とする。初期構築では管理者1名とする。

## 3. 機能要件

### 3.1 LP・公開ページ

| ID | 要件 |
|---|---|
| PUB-01 | 日本語、英語、簡体中国語を切り替えられる |
| PUB-02 | 価値、利用例、仕組み、対応言語、料金、FAQを掲載する |
| PUB-03 | 45〜60秒の実演動画を掲載できる |
| PUB-04 | 無料登録とログインへの導線を常時表示する |
| PUB-05 | 献金について表示しない |
| PUB-06 | 対応言語・料金・免責を誤認のない表現で掲載する |

### 3.2 会員登録・認証

| ID | 要件 |
|---|---|
| AUTH-01 | メールアドレスとパスワードで登録できる |
| AUTH-02 | メール確認が完了するまで通訳と無料付与を利用できない |
| AUTH-03 | パスワード再設定メールを送信できる |
| AUTH-04 | ログイン失敗を回数制限し、総当たり攻撃を抑止する |
| AUTH-05 | 利用規約とプライバシーポリシーへの同意日時・版を保存する |
| AUTH-06 | 退会申請と個人データ削除を行える |
| AUTH-07 | 18歳未満は保護者同意が必要である旨を表示する |

### 3.3 初回無料枠

| ID | 要件 |
|---|---|
| TRIAL-01 | メール確認完了時に900秒を一度だけ付与する |
| TRIAL-02 | 登録から30日で未使用分を失効させる |
| TRIAL-03 | 無料秒数から先に消費する |
| TRIAL-04 | 同一人物による反復取得の疑いを検知・制限する |
| TRIAL-05 | 無料枠を譲渡、換金、返金できない |

### 3.4 時間商品・決済

| ID | 要件 |
|---|---|
| PAY-01 | 固定された時間商品を購入できる |
| PAY-02 | Stripe Checkoutへ遷移して一回払いできる |
| PAY-03 | 決済完了は署名検証済みWebhookだけで確定する |
| PAY-04 | 同一Stripeイベントを二重処理しない |
| PAY-05 | 購入時間、金額、通貨、税、決済状態を履歴表示する |
| PAY-06 | 返金・チャージバック時は未使用時間を安全に調整する |
| PAY-07 | 残高・履歴は利用者が改ざんできないサーバーDBで管理する |
| PAY-08 | 有料時間の有効期限は購入日から180日を初期値とする |

### 3.5 リアルタイム通訳

| ID | 要件 |
|---|---|
| INT-01 | 入力言語と出力言語を選択できる |
| INT-02 | 入出力が同じ言語の組合せを開始できない |
| INT-03 | 利用可能なマイクを選択できる |
| INT-04 | 翻訳開始前からマイク入力レベルを常時表示する |
| INT-05 | 通訳音声と同程度の基準音を再生して音量確認できる |
| INT-06 | 字幕や文字起こしを表示しない |
| INT-07 | 開始、一時停止、再開、終了ができる |
| INT-08 | 出力音量の調整とミュートができる |
| INT-09 | 接続状態、残り時間、今回利用時間を表示する |
| INT-10 | サーバー側で秒単位に利用時間を計測・差し引く |
| INT-11 | 残高0秒でAPI接続をサーバー側から終了する |
| INT-12 | 通信断時に限定回数で再接続し、二重課金しない |
| INT-13 | マイク拒否、残高不足、API障害を利用言語で案内する |
| INT-14 | 1セッションは最大120分とし、継続には再接続を求める |

### 3.6 マイページ

- 無料・有料・合計の残り時間
- 購入ボタンと商品一覧
- 購入履歴、利用履歴
- 言語・音量設定
- メール変更、パスワード変更
- ログアウト、退会

### 3.7 管理画面

- 会員検索、停止、退会状態
- 無料枠付与状況と不正兆候
- 残高、台帳、手動調整
- Stripe決済、返金、チャージバック
- 通訳セッション、利用秒数、言語ペア、エラー
- 売上、Stripe手数料、API原価概算、粗利益概算
- 商品、料金、対応言語、保守状態
- 操作監査ログ

## 4. クレジット会計要件

- 内部単位は秒とする。
- 残高を直接上書きせず、付与・消費・失効・返金・調整を追記型台帳に記録する。
- 各取引に一意な冪等キーを持たせる。
- 無料残高と有料残高を分離する。
- 通訳開始時に短い時間枠を予約し、終了時に実利用分と精算する。
- 残高が負数にならないDB制約とトランザクションを用いる。

## 5. 多言語要件

- 初期UI: `ja`、`en`、`zh-CN`
- URL、ブラウザ設定、会員設定の順に表示言語を決定する。
- 翻訳対象言語の一覧はサーバー設定から配信し、API仕様変更に追従可能にする。
- 法定ページも3言語で用意するが、日本法上の正本は日本語版と明記する。
- 日付、金額、通貨、数値はロケールに合わせて表示する。

## 6. 非機能要件

### 対応環境

- PC: Chrome、Edge、Safariの現行主要版
- スマートフォン: iOS Safari、Android Chromeの現行主要版
- HTTPSとマイク権限が利用できる環境

### 性能

- LP主要表示: 通常回線で3秒以内を目標
- API以外の画面操作: 95パーセンタイル1秒以内を目標
- 通訳遅延: API・通信条件に依存するため保証値ではなく実測値を監視

### セキュリティ

- HTTPS強制、Secure/HttpOnly/SameSite Cookie
- CSRF、XSS、SQLインジェクション、総当たり対策
- パスワードはPHP `password_hash` の安全な方式で保存
- Stripe署名検証、Webhook冪等処理
- APIキー、DB・FTP・Stripe秘密鍵はGitへ保存しない
- 管理操作は監査ログへ記録
- 音声や翻訳本文をアプリログへ記録しない

### 可用性・バックアップ

- 初期はベストエフォートとしSLAを約束しない
- DBは日次バックアップ、30日保持を目標
- 障害時は決済と残高の整合性を優先して通訳開始を停止する
- Stripe決済と残高台帳を日次照合できる

## 7. データ保持

| データ | 保持方針 |
|---|---|
| 音声・翻訳本文 | 保存しない |
| 会員情報 | 利用中および退会後の法定・不正対応期間 |
| 決済・残高台帳 | 法令・会計要件に合わせて保持 |
| セッションメタデータ | 180日を初期値 |
| セキュリティログ | 90日を初期値 |
| メール認証・再設定トークン | 有効期限後に削除 |

## 8. 法定・運用ページ

- 利用規約
- プライバシーポリシー
- 特定商取引法に基づく表記
- 返金・キャンセルポリシー
- 通訳時間・無料体験の利用条件
- AI通訳に関する免責と禁止用途
- Cookieポリシー
- 問い合わせ窓口

販売者氏名、住所、電話番号、問い合わせメール等は本番公開前にAsaphが確認する。ソースコードには直接書かず、本番設定または管理画面から設定する。

## 9. 受入条件

- メール認証済み新規会員だけに900秒が一度だけ付与される。
- Stripeのテスト決済で商品時間が一度だけ付与される。
- 再送Webhookでも二重付与されない。
- 残高0で通訳を開始できず、利用中に0になると停止する。
- 無料・有料残高の消費順と失効が正しい。
- 3言語UIで主要導線が欠けない。
- 音声と翻訳本文がDB・ログへ残らない。
- 一般利用者が管理APIを操作できない。
- 主要端末でマイク、入力メーター、テスト音、通訳が動作する。
- 本番秘密情報がGit履歴とブラウザ配信物に存在しない。

## 10. Asaph本人にしか完了できない項目

- Stripeアカウントの本人確認
- Stripeの入金銀行口座登録
- 販売者の法定氏名、住所、電話番号の確認・入力
- 本番OpenAI APIキーとStripe本番秘密鍵の発行・入力
- 本番販売開始の最終承認

これら以外の設計、実装、テスト、デプロイ準備はCodexが進める。
