# ライブ通訳サービス（事業版）基本設計書

作成日: 2026-07-11  
工程: 基本設計  
状態: 初版

## 1. 全体構成

- フロントエンド: React、TypeScript、Vite
- Web API: PHP 8系
- DB: MySQL
- Webホスティング: ConoHa WING
- 決済: Stripe Checkout、Stripe Webhook
- 通訳: OpenAI Realtime Translation API
- リアルタイム中継: Node.js WebSocketサービス（配置先は負荷・費用試験後に確定）
- メール: トランザクションメールサービスまたはConoHa SMTP（到達率試験で選定）
- ソース管理: GitHub非公開リポジトリ

## 2. コンポーネント責務

| コンポーネント | 責務 |
|---|---|
| React UI | LP、認証、マイページ、通訳画面、管理画面、多言語表示 |
| PHP API | 認証、商品、決済開始、Webhook、残高、履歴、管理API |
| MySQL | 会員、同意、商品、決済、残高台帳、セッション、監査 |
| Realtime Gateway | 短時間の利用許可、音声中継、秒数計測、残高0で切断 |
| Stripe | カード情報の受付、決済、返金、チャージバック通知 |
| OpenAI | 音声から音声へのリアルタイム翻訳 |

## 5. 通訳・残高フロー

1. 会員が言語とマイクを選択する。
2. PHP APIがログイン、残高、利用停止状態を確認する。
3. APIが短時間・一回限りのGateway接続許可を発行する。
4. Gatewayが許可を検証し、OpenAIへ接続する。
5. Gatewayが利用秒数を計測し、短い間隔で利用枠を予約する。
6. 無料残高、有料残高の順に台帳へ消費を記録する。
7. 残高0、最大時間、異常検知でGatewayが接続を閉じる。
8. 終了時に予約分と実利用分を精算する。

## 6. 決済フロー

1. 会員がサーバーから取得した商品を選択する。
2. PHP APIがStripe Checkout Sessionを作成する。
3. Stripe画面で決済する。
4. Stripe Webhookを署名検証する。
5. イベントIDを一意登録する。
6. DBトランザクション内で決済確定と有料時間付与を行う。
7. 画面はサーバーから最新残高を再取得する。

## 7. セキュリティ境界

- カード番号は本システムへ送信・保存しない。
- OpenAIとStripeの秘密鍵はサーバーだけが保持する。
- ブラウザへ長期APIキーを渡さない。
- 残高判定と利用時間確定をブラウザに任せない。
- 管理画面は別権限、二要素認証、監査ログを必須とする。
- 音声と翻訳本文はDBや通常ログへ保存しない。

## 8. デプロイ境界

- `main`への承認済みpushでConoHaへ自動デプロイする。
- `api/config.php`、`.env`、秘密鍵はデプロイ対象外とする。
- DB変更は番号付きマイグレーションで適用する。
- Realtime GatewayはWeb本体とは独立してデプロイ・停止できる。
